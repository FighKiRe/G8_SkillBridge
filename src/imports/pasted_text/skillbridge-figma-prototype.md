Create a modern desktop web application UI prototype in Figma called “SkillBridge” focused on SDG 8: Decent Work and Economic Growth. The platform helps fresh graduates and NEET youth gain practical experience through real-world micro-projects posted by companies. Users can complete projects, receive employer feedback, earn certificates and badges, and build a digital portfolio even without formal work experience.

The platform should help solve unemployment, underemployment, skills mismatch, and lack of work experience among fresh graduates.

DESIGN STYLE:

* Modern SaaS dashboard UI
* Clean, minimalist, and professional
* Inspired by LinkedIn, Upwork, Coursera, Stripe, and Notion
* Youth-friendly but professional
* Spacious layout with strong visual hierarchy
* Soft shadows and rounded corners
* Clean white/light gray backgrounds
* Professional illustrations and modern icons

IMPORTANT UI/UX REQUIREMENTS:

* Follow Norman Design Principles:
  visibility,
  feedback,
  consistency,
  affordance,
  simplicity,
  accessibility,
  and user-centered design.
* Every screen must feel visually connected.
* Avoid clutter and overcrowded layouts.
* Use modern dashboard spacing and alignment.
* All buttons, cards, forms, and tags should be reusable components.
* Include realistic notification badges and active states.
* Add hover-ready button designs and realistic dashboard interactions.
* Notifications should visibly appear with unread indicators.

IMPORTANT AUTHENTICATION FLOW:

* During registration:

  * Separate input fields for:
    First Name
    Last Name
  * Include:
    Email
    Password
    Confirm Password
  * Add password visibility toggle icon
  * Add password validation indicators
  * Add clean error/success validation states

* During login:

  * Users should NOT immediately enter employer or graduate dashboards.
  * After successful login or registration, users should first choose or be redirected based on their account role:
    Graduate
    Employer

* Create separate flows and dashboards for:

  * Graduate users
  * Employer users

* The system should clearly distinguish employer-side and graduate-side experiences after authentication.
